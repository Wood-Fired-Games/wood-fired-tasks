# Navigation: task-oriented file map

Owner: Repository maintainers

Task-oriented index. For each common change request, lists the **implementation files**, **tests to update**, and **docs to check** so an agent can start editing in under one screen of reading. Assumes you have already scanned the *Read-next by intent* table in [`AGENTS.md`](../AGENTS.md). Deeper context: [`docs/AGENT_CONTEXT.md`](AGENT_CONTEXT.md) (contract), [`docs/REPO_MAP.md`](REPO_MAP.md) (tree), [`docs/ARCHITECTURE.md`](ARCHITECTURE.md) (data flow), [`docs/INTERFACES.md`](INTERFACES.md) (surface inventory).

## Source vs generated

Source-of-truth code is in `src/`. Everything else is regenerated or installed:

- `dist/` â€” `tsc` build output, regenerated by `npm run build`. Never hand-edit.
- `bin/tasks.cmd` â€” Windows shim baked by `client-package/setup.bat` / `setup.ps1` at install time; carries a baked-in API key and is **not committed**. Distinguish from `src/cli/bin/tasks.ts`, which *is* source.
- `.agent-context.json`, `dependency-graph.dot`, `coverage/`, `reports/` â€” generated data and reports. Regenerate via the matching `npm run` script; do not hand-edit.
- `package-lock.json` is committed but managed by `npm`.

## Change-type recipes

### 1. Add or change an API route

- **Files:** `src/api/routes/<resource>/index.ts` (or `src/api/routes/{events,health}.ts`), matching `src/api/routes/<resource>/schemas.ts`, the underlying `src/services/<resource>.service.ts`.
- **Tests:** `src/api/__tests__/<resource>.test.ts`, `scripts/agent-context/__tests__/interfaces-counts.test.ts`.
- **Docs:** `docs/API.md`, `docs/INTERFACES.md`, `docs/AGENT_CONTEXT.md`.

### 2. Add or change an MCP tool

- **Files:** `src/mcp/tools/<area>-tools.ts` (one of `comment|dependency|health|project|task`), the matching `src/services/<area>.service.ts`, `src/mcp/server.ts` if registration changes.
- **Tests:** `src/mcp/__tests__/<area>-tools.test.ts`, `scripts/agent-context/__tests__/interfaces-counts.test.ts`.
- **Docs:** `docs/MCP.md`, `docs/INTERFACES.md`, `docs/AGENT_CONTEXT.md`.

### 3. Change remote MCP proxy behavior

- **Files:** `src/mcp/remote/index.ts`, `src/mcp/remote/register-tools.ts`, `src/mcp/remote/rest-client.ts`. Required env: `WFT_API_URL`, `WFT_API_KEY`.
- **Tests:** `src/mcp/__tests__/remote-*.test.ts`, `src/mcp/__tests__/register-tools.test.ts`.
- **Docs:** `docs/MCP.md`, `docs/SETUP.md`, `docs/AGENT_CONTEXT.md`.

### 4. Add or change a CLI command

- **Files:** `src/cli/bin/tasks.ts` (Commander entry / registration), `src/cli/commands/<name>.ts`, `src/cli/api/client.ts` if a new REST call is needed.
- **Tests:** `src/cli/__tests__/<name>.test.ts`, `scripts/agent-context/__tests__/interfaces-counts.test.ts`.
- **Docs:** `docs/CLI.md`, `docs/INTERFACES.md`.

### 5. Add or change a schema field

- **Files:** `src/schemas/{task,comment,dependency,idempotency}.schema.ts`. A field change ripples through services, repositories, and every surface that imports the schema â€” search consumers before editing.
- **Tests:** `src/schemas/__tests__/`, plus any service/repository/surface test that constructs the affected shape.
- **Docs:** `docs/API.md`, `docs/INTERFACES.md`, `docs/MCP.md` / `docs/CLI.md` if the field is user-visible on those surfaces.

### 6. Add or change a repository method

- **Files:** `src/repositories/{task,comment,project,dependency}.repository.ts`, `src/repositories/interfaces.ts` if the contract changes, `src/repositories/row-mapper.ts` if the DB shape moves.
- **Tests:** `src/repositories/__tests__/<resource>.repository.test.ts`.
- **Docs:** `docs/ARCHITECTURE.md` if data flow shifts; usually no surface-doc change.

### 7. Change a service invariant

- **Files:** `src/services/<area>.service.ts` (the one that owns the invariant), `src/services/workflow-engine.ts` if it spans status / lifecycle, `src/services/errors.ts` for new error variants.
- **Tests:** `src/services/__tests__/<area>.service.test.ts`, plus any property test in `src/services/__tests__/` that asserts the invariant.
- **Docs:** `docs/ARCHITECTURE.md`, `docs/AGENT_CONTEXT.md` if the contract for surfaces changes.

### 8. Add a DB migration

### 18. Break down a project goal into tasks

- **Design / contract:** [`docs/tasks-decompose-design.md`](tasks-decompose-design.md)
  (source of truth â€” guardrails, artifact schema, cost budget).
- **Schema:** [`src/lib/decompose/schema.ts`](../src/lib/decompose/schema.ts)

### 19. Harden loop evidence against fabrication

If you want to stop `/tasks:loop` / `/tasks:loop-dag` agents from closing tasks
on fabricated evidence (nonexistent git SHAs, self-grading, placeholder
verifier text), there are three defense-in-depth layers â€” read
[`docs/RELIABILITY.md`](RELIABILITY.md) first for the full picture and an honest
statement of what each layer does **not** guarantee.

- **Server-side gate (Piece A):** `src/services/evidence-validation.ts` (pure
  fn), wired into `update_task` via `src/services/task.service.ts`, gated behind
  the `WFT_STRICT_EVIDENCE` env flag (**default off** â€” opt in per
  [`docs/SETUP.md`](SETUP.md) â†’ *Optional hardening flags*).
- **Client-side hook (Piece B):** [`docs/hooks/validate-sha.mjs`](hooks/validate-sha.mjs)
  + [`docs/hooks/README.md`](hooks/README.md) â€” an optional `PreToolUse` guard
  that blocks evidence citing SHAs unknown to the local repo. Client-side by
  necessity: the server cannot reach an arbitrary client's checkout.
- **Skill discipline (Piece C):** the anti-fabrication clause,
  one-state-mutation-per-turn rule, and separate-verifier requirement in
  [`skills/tasks/loop-shared.md`](../skills/tasks/loop-shared.md) Â§L, referenced
  from [`loop.md`](../skills/tasks/loop.md) / [`loop-dag.md`](../skills/tasks/loop-dag.md).
- **Tests:** `src/services/__tests__/strict-evidence-validation.test.ts`,
  `src/__tests__/validate-sha-hook.test.ts`.
- **Docs:** `docs/RELIABILITY.md`, `docs/hooks/README.md`, `docs/SETUP.md`.

### 20. Change WSJF scoring / ranking / charter behavior

WSJF (Weighted Shortest Job First) scores every task on Cost of Delay
(Business Value + Time Criticality + Risk/Opportunity) Ă· Job Size and spans the
full surface stack. The LLM only emits classifications over closed enums +
verbatim evidence; the **server recomputes every Fibonacci component
deterministically** (`validateScoreSubmission`). Identify the layer you are
touching before editing â€” the scoring math, the read-time ranking, the rescore
transaction, the health linter, or one of the REST/MCP/CLI adapters.

- **Files (services â€” the source of truth, surfaces are thin adapters):**
  - `src/services/wsjf.service.ts` â€” the deterministic gate
    (`validateScoreSubmission`, `validateManualScore`), `computeWsjf`, and the
    read-time `rankFrontier` (blocker-propagated effective WSJF; Îł=0.5, cap 3Ă—).
  - `src/services/wsjf-rescore.service.ts` â€” `WsjfRescoreService`: re-runs the
    gate against the current charter in one transaction, skips locked
    components, writes one history row per changed task.
  - `src/services/wsjf-health.service.ts` â€” `WsjfHealthService` /
    `analyzeWsjfHealth`: non-blocking degeneracy linter (six checks).
  - `src/repositories/wsjf-history.repository.ts` â€” `WsjfHistoryRepository`,
    the append-only owner of the audit tables (INSERT-only; no UPDATE/DELETE).
- **Files (REST routes):** `src/api/routes/tasks/wsjf.ts` (GET/PUT
  `/api/v1/tasks/:id/wsjf`, GET `/api/v1/tasks/:id/score-history`),
  `src/api/routes/projects/wsjf.ts` (GET `/charter-history`, GET
  `/rescore-runs`, GET `/wsjf-ranking`, GET `/wsjf-health`, POST `/rescore`
  under `/api/v1/projects/:id`). Route mounting in `src/api/server.ts`.
- **Files (MCP tools):** `src/mcp/tools/wsjf-tools.ts` (stdio: `wsjf_ranking`,
  `wsjf_history`, `rescore_project`, `wsjf_health`), registered in
  `src/mcp/server.ts`; `src/mcp/remote/register-tools.ts` (remote proxies, with
  `src/mcp/remote/rest-client.ts`). The four tools have **full stdioâ†”remote
  parity** â€” change both transports together.
- **Files (CLI):** `src/cli/commands/wsjf.ts` (`tasks wsjf-history`,
  `tasks wsjf-set`, `tasks charter-history`), registered in

- **Files (charter on project):** the `value_charter` field on `create_project`

- **Tests:** the WSJF service suite under `src/services/__tests__/`

## Cross-cutting reminders

- Schemas in `src/schemas/` are imported by **every** surface â€” a one-line change ripples to REST, MCP, CLI types simultaneously. Search consumers before editing.
- Shared behavior belongs in **services**; routes, MCP tools, CLI commands, and Slack handlers are thin adapters. Resist adding logic to a surface.
- When a route, tool, command, or migration is added, update `docs/INTERFACES.md` and re-run `npm test` so the drift-detection test (`scripts/agent-context/__tests__/interfaces-counts.test.ts`) passes.
- Vendor-specific files (`.claude/`, `CLAUDE.md`, `.cursor/`, `.gemini/`, `.codex/`) **must not** carry unique facts â€” see [`docs/AGENT_CONTEXT.md`](AGENT_CONTEXT.md) Â§6. Move any normative content back into the authoritative tier.
- After any canonical-doc edit, run `npm run agent-context:check` and (if the manifest changed) `npm run agent-context:gen`.
