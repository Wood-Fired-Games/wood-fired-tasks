# Navigation: task-oriented file map

Owner: Repository maintainers

Task-oriented index. For each common change request, lists the **implementation files**, **tests to update**, and **docs to check** so an agent can start editing in under one screen of reading. Assumes you have already scanned the *Read-next by intent* table in [`AGENTS.md`](../AGENTS.md). Deeper context: [`docs/AGENT_CONTEXT.md`](AGENT_CONTEXT.md) (contract), [`docs/REPO_MAP.md`](REPO_MAP.md) (tree), [`docs/ARCHITECTURE.md`](ARCHITECTURE.md) (data flow), [`docs/INTERFACES.md`](INTERFACES.md) (surface inventory).

## Source vs generated

Source-of-truth code is in `src/`. Everything else is regenerated or installed:

- `dist/` — `tsc` build output, regenerated by `npm run build`. Never hand-edit.
- `bin/tasks.cmd` — Windows shim baked by `client-package/setup.bat` / `setup.ps1` at install time; carries a baked-in API key and is **not committed**. Distinguish from `src/cli/bin/tasks.ts`, which *is* source.
- `.agent-context.json`, `dependency-graph.dot`, `coverage/`, `reports/` — generated data and reports. Regenerate via the matching `npm run` script; do not hand-edit.
- `package-lock.json` is committed but managed by `npm`.

## Change-type recipes

### 1. Add or change an API route

- **Files:** `src/api/routes/<resource>/index.ts` (or `src/api/routes/{events,health}.ts`), matching `src/api/routes/<resource>/schemas.ts`, the underlying `src/services/<resource>.service.ts`.
- **Tests:** `src/api/__tests__/<resource>.test.ts`, `scripts/agent-context/__tests__/interfaces-counts.test.ts`.
- **Docs:** `docs/API.md`, `docs/INTERFACES.md`, `docs/AGENT_CONTEXT.md`.

### 2. Add or change an MCP tool

- **Files:** `src/mcp/tools/<area>-tools.ts` (one of `comment|dependency|health|project|task`), the matching `src/services/<area>.service.ts`, `src/mcp/server.ts` if registration changes.
- **Tests:** `src/mcp/__tests__/<area>-tools.test.ts`, `scripts/agent-context/__tests__/interfaces-counts.test.ts`.
- **Docs:** `docs/MCP.md`, `docs/INTERFACES.md`, `docs/AGENT_CONTEXT.md`.

### 3. Change remote MCP proxy behavior

- **Files:** `src/mcp/remote/index.ts`, `src/mcp/remote/register-tools.ts`, `src/mcp/remote/rest-client.ts`. Required env: `WFB_API_URL`, `WFB_API_KEY`.
- **Tests:** `src/mcp/__tests__/remote-*.test.ts`, `src/mcp/__tests__/register-tools.test.ts`.
- **Docs:** `docs/MCP.md`, `docs/SETUP.md`, `docs/AGENT_CONTEXT.md`.

### 4. Add or change a CLI command

- **Files:** `src/cli/bin/tasks.ts` (Commander entry / registration), `src/cli/commands/<name>.ts`, `src/cli/api/client.ts` if a new REST call is needed.
- **Tests:** `src/cli/__tests__/<name>.test.ts`, `scripts/agent-context/__tests__/interfaces-counts.test.ts`.
- **Docs:** `docs/CLI.md`, `docs/INTERFACES.md`.

### 5. Add or change a schema field

- **Files:** `src/schemas/{task,comment,dependency,idempotency}.schema.ts`. A field change ripples through services, repositories, and every surface that imports the schema — search consumers before editing.
- **Tests:** `src/schemas/__tests__/`, plus any service/repository/surface test that constructs the affected shape.
- **Docs:** `docs/API.md`, `docs/INTERFACES.md`, `docs/MCP.md` / `docs/CLI.md` if the field is user-visible on those surfaces.

### 6. Add or change a repository method

- **Files:** `src/repositories/{task,comment,project,dependency}.repository.ts`, `src/repositories/interfaces.ts` if the contract changes, `src/repositories/row-mapper.ts` if the DB shape moves.
- **Tests:** `src/repositories/__tests__/<resource>.repository.test.ts`.
- **Docs:** `docs/ARCHITECTURE.md` if data flow shifts; usually no surface-doc change.

### 7. Change a service invariant

- **Files:** `src/services/<area>.service.ts` (the one that owns the invariant), `src/services/workflow-engine.ts` if it spans status / lifecycle, `src/services/errors.ts` for new error variants.
- **Tests:** `src/services/__tests__/<area>.service.test.ts`, plus any property test in `src/services/__tests__/` that asserts the invariant.
- **Docs:** `docs/ARCHITECTURE.md`, `docs/AGENT_CONTEXT.md` if the contract for surfaces changes.

### 8. Add a DB migration

- **Files:** new `src/db/migrations/NNN-<slug>.ts` (umzug; both `up` and `down`), `src/db/migrate.ts` only if migration plumbing changes, `src/repositories/*` and `src/schemas/*` to consume the new columns.
- **Tests:** `src/db/__tests__/migrations-roundtrip.test.ts`, `src/db/__tests__/<slug>.test.ts` (new).
- **Docs:** `docs/ARCHITECTURE.md`, `docs/SETUP.md` if the migration changes setup instructions.

### 9. Change Slack slash command behavior

- **Files:** `src/slack/commands/tasks-command.ts` (single dispatcher with sub-routes), `src/slack/user-identity.ts`, supporting `src/services/*` the command calls.
- **Tests:** `src/slack/__tests__/tasks-command*.test.ts`.
- **Docs:** `docs/SLACK.md`, `slack-app-manifest.yml` if scopes / command surface changes.

### 10. Change Slack outbound notifier behavior

- **Files:** `src/slack/notifier.ts`, `src/slack/task-formatter.ts`, `src/services/slack.service.ts`, `src/slack/repositories/channel-subscription.repository.ts` for routing changes.
- **Tests:** `src/slack/__tests__/notifier.test.ts`, `src/slack/__tests__/task-formatter.test.ts`.
- **Docs:** `docs/SLACK.md`.

### 11. Add or change event emission

- **Files:** `src/events/event-bus.ts`, `src/events/types.ts` (add to `ALLOWED_EVENT_TYPES`), `src/events/sse-manager.ts` if subscription semantics shift. Emissions live in the relevant `src/services/<area>.service.ts`. SSE endpoint: `src/api/routes/events.ts`.
- **Tests:** `src/events/__tests__/event-bus.test.ts`, `src/events/__tests__/sse-manager.test.ts`, `src/api/__tests__/events.test.ts`.
- **Docs:** `docs/API.md` (SSE section), `docs/ARCHITECTURE.md`, `docs/INTERFACES.md`.

### 12. Change completion report behavior

- **Files:** `src/mcp/tools/task-tools.ts` (the `completion_report` tool handler), `src/services/task.service.ts` (underlying implementation), `src/schemas/task.schema.ts` if the payload shape changes.
- **Tests:** `src/mcp/__tests__/task-tools.test.ts`, `src/services/__tests__/task.service.test.ts`.
- **Docs:** `docs/MCP.md`, `docs/INTERFACES.md`.

### 13. Change dependency behavior (add / remove edges, cycle detection)

- **Files:** `src/services/dependency.service.ts`, the `CycleDetector` helper imported there, `src/repositories/dependency.repository.ts`, `src/api/routes/dependencies/index.ts`, `src/mcp/tools/dependency-tools.ts`.
- **Tests:** `src/services/__tests__/dependency.service.test.ts` (including cycle-detection cases), `src/repositories/__tests__/dependency.repository.test.ts`.
- **Docs:** `docs/API.md`, `docs/MCP.md`.

### 14. Change task status transition rules

- **Files:** `src/types/task.ts` (`VALID_STATUS_TRANSITIONS`), `src/services/task.service.ts`, `src/services/workflow-engine.ts`.
- **Tests:** `src/services/__tests__/task.service.test.ts`, `src/services/__tests__/workflow-engine.test.ts`.
- **Docs:** `docs/ARCHITECTURE.md`, `docs/API.md`, `docs/MCP.md` (status enum is user-visible).

### 15. Change installer behavior

- **Files:** `install.sh`, `install.ps1` (repo root), `client-package/setup.sh`, `client-package/setup.bat`, `client-package/setup.ps1`, `client-package/commands/`. Note: `bin/tasks.cmd` is **produced** by these scripts on install — do not edit it as source.
- **Tests:** `.github/workflows/install-scripts.yml` is the integration gate; some helpers have unit tests in `scripts/__tests__/`.
- **Docs:** `docs/SETUP.md`, `README.md` (quickstart).

### 16. Make a docs-only change

- **Files:** the doc you are editing (`docs/*.md`, `README.md`, `CONTRIBUTING.md`, `AGENTS.md`). Confirm ownership and budget in `docs/AGENT_CONTEXT.md`.
- **Tests:** none if prose-only; `scripts/agent-context/__tests__/` if you touch a canonical doc, line budget, or manifest entry.
- **Docs:** `docs/AGENT_CONTEXT.md` (contract), and re-run `npm run agent-context:check` to verify budgets / ownership.

### 17. Change release / packaging metadata

- **Files:** `package.json` (`scripts.prepublishOnly`, `version`, `bin`, `files`), `CHANGELOG.md`, `.github/workflows/*.yml`, `client-package/` if the install bundle changes.
- **Tests:** existing release-related tests under `scripts/__tests__/`; CI workflow is the integration gate.
- **Docs:** `docs/RELEASE.md`, `README.md` install section, `CONTRIBUTING.md`.

## Cross-cutting reminders

- Schemas in `src/schemas/` are imported by **every** surface — a one-line change ripples to REST, MCP, CLI types simultaneously. Search consumers before editing.
- Shared behavior belongs in **services**; routes, MCP tools, CLI commands, and Slack handlers are thin adapters. Resist adding logic to a surface.
- When a route, tool, command, or migration is added, update `docs/INTERFACES.md` and re-run `npm test` so the drift-detection test (`scripts/agent-context/__tests__/interfaces-counts.test.ts`) passes.
- Vendor-specific files (`.claude/`, `CLAUDE.md`, `.cursor/`, `.gemini/`, `.codex/`) **must not** carry unique facts — see [`docs/AGENT_CONTEXT.md`](AGENT_CONTEXT.md) §6. Move any normative content back into the authoritative tier.
- After any canonical-doc edit, run `npm run agent-context:check` and (if the manifest changed) `npm run agent-context:gen`.
