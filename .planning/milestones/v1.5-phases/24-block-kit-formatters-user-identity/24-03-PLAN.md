**Class structure:**
```typescript
import type { WebClient } from '@slack/web-api';

interface CacheEntry {
  displayName: string;
  expiresAt: number;
}

export class UserIdentityCache {
  private readonly cache = new Map<string, CacheEntry>();

  constructor(
    private readonly client: WebClient,
    private readonly ttlMs: number = 5 * 60 * 1000
  ) {}

  async resolve(userId: string): Promise<string> { ... }
  clear(): void { this.cache.clear(); }
}
```

**resolve() implementation:**
1. Check cache: if entry exists and `expiresAt > Date.now()`, return cached `displayName`
2. Try `this.client.users.info({ user: userId })`
3. Extract display name with fallback chain: `profile.display_name` (trimmed, non-empty) → `profile.real_name` (trimmed, non-empty) → `user.name` → `userId`
4. Cache result with `expiresAt = Date.now() + this.ttlMs`
5. On error: cache `userId` with short TTL (30,000ms), return `userId`

**No logger parameter** — keep the class minimal. Errors are handled gracefully (return userId). Phase 25 handler can log if needed before calling resolve().

**Key design choices:**
- Constructor takes `WebClient` (not `SlackService`) for dependency inversion and testability
- TTL is configurable via constructor (default 5 min)
- Error TTL is hardcoded at 30s (internal constant `ERROR_TTL_MS = 30_000`)
- No pino dependency — this class is self-contained
