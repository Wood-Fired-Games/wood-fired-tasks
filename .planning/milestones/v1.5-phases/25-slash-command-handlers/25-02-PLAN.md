Implement 6 handler functions and wire them into the existing switch statement (replacing the 'not yet implemented' stubs):

**1. handleList(respond, services, args):**
- Parse args with `parseArgs(args)` to extract flags: `--status`, `--project`, `--assignee`, `--search`, `--tags`
- Build filters object: `{ status: flags.status, project_id: flags.project ? parseInt(flags.project, 10) : undefined, assignee: flags.assignee, search: flags.search, tags: flags.tags ? flags.tags.split(',') : undefined }`
- Call `services.taskService.listTasks(filters)`
- Call `formatTaskList(tasks)` to get blocks
- Call `respondBlocks(respond, blocks, 'Tasks (' + tasks.length + ')')`

**2. handleShow(respond, services, args):**
- Parse id from `args[0]`: `const id = parseInt(args[0] ?? '', 10)`. If NaN, respondError with usage hint.
- Call `services.taskService.getTask(id)` for task detail
- Call `formatTaskDetail(task)` for base blocks
- Call `services.commentService.getComments(id)` — append last 5 comments as SectionBlocks (author + content), preceded by a DividerBlock and a SectionBlock header "*Comments*". If more than 5, add ContextBlock footer "X more comments".
- Call `services.dependencyService.getBlockedBy(id)` and `services.dependencyService.getBlockers(id)` — append as a SectionBlock with mrkdwn listing "Blocks: #X, #Y" and "Blocked by: #A, #B", preceded by a DividerBlock. Skip if both empty.
- Call `respondBlocks(respond, allBlocks, 'Task #' + id)`

**3. handleCreate(respond, services, identityCache, command, args):**
- Parse args with parseArgs — positionals become the title (joined with spaces: `positionals.join(' ')`). Flags: `--project` (required), `--priority` (optional, default 'medium'), `--description` (optional).
- If no title (positionals empty), respondError with usage: `/tasks create <title> --project <id>`
- If no --project flag, respondError: "Project ID required. Usage: `/tasks create <title> --project <id>`"
- Resolve display name: `const createdBy = await identityCache.resolve(command.user_id)` (UIDENT-03)
- Call `services.taskService.createTask({ title, project_id: parseInt(flags.project, 10), priority: flags.priority || 'medium', created_by: createdBy, description: flags.description || null })`
- Format with `formatTaskDetail(task)` and respond

**4. handleUpdate(respond, services, args):**
- First arg is task ID: `const id = parseInt(args[0] ?? '', 10)`. If NaN, respondError.
- Parse remaining args (args.slice(1)) with parseArgs for flags: `--status`, `--title`, `--priority`, `--assignee`, `--due`, `--description`
- Build updates object from present flags: `{ status: flags.status, title: flags.title, priority: flags.priority, assignee: flags.assignee, due_date: flags.due, description: flags.description }`
- Remove undefined keys (only pass fields that were explicitly provided)
- If no flags present, respondError: "No update fields provided."
- Call `services.taskService.updateTask(id, updates)`
- Format with `formatTaskDetail(task)` and respond

**5. handleDelete(respond, services, args):**
- Parse id from args[0]. If NaN, respondError.
- Call `services.taskService.deleteTask(id)`
- Respond with simple SectionBlock: ":white_check_mark: Task #<id> deleted."

**6. handleClaim(respond, services, identityCache, command, args):**
- Parse id from args[0]. If NaN, respondError with usage hint.
- Resolve display name: `const displayName = await identityCache.resolve(command.user_id)` (UIDENT-03)
- Call `services.taskService.claimTask(id, displayName)`
- Format with `formatTaskDetail(task)` and respond with fallback text `Task #${id} claimed by ${displayName}`

Wire all handlers into the switch statement, replacing their stub cases. Pass appropriate arguments (respond, services, identityCache, command, args) to each handler.
