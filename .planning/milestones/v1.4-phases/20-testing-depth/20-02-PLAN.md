       ```typescript
       import { describe } from 'vitest';
       import { CycleDetector } from '../cycle-detector.js';
       ```

       Properties to test:

       a) **Empty graph, no cycles:** For any two distinct nodes in an empty graph, `wouldCreateCycle(a, b)` returns false.

          - Property: `new CycleDetector([]).wouldCreateCycle(a, b) === false`

       b) **Self-loop detection:** For any node, `wouldCreateCycle(a, a)` returns true (adding a->a is always a cycle).
          - Arbitrary: single integer
          - Property: `new CycleDetector([]).wouldCreateCycle(a, a) === true`

       c) **Return type invariant:** For any graph and any proposed edge, `wouldCreateCycle` always returns a boolean (never throws).
          - Arbitrary: array of edges (up to 20), two integers
          - Filter: edges have distinct task_id and blocks_task_id
          - Property: `typeof detector.wouldCreateCycle(from, to) === 'boolean'`

       d) **Mutual edge creates cycle:** If edge A->B exists, then `wouldCreateCycle(B, A)` must return true (B->A would close the cycle A->B->A).
          - Arbitrary: two distinct integers
          - Property: given edge `{task_id: a, blocks_task_id: b}`, `wouldCreateCycle(b, a) === true`

       Use `fc.pre(a !== b)` for preconditions where nodes must be distinct. Use `fc.integer({ min: 1, max: 100 })` for node IDs. Use `fc.array(edgeArb, { maxLength: 20 })` for edge lists.
