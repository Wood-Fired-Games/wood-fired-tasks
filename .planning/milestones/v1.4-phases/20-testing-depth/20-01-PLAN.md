<task type="auto">
  <name>Task 1: Install knip and configure with entry points and exclusions</name>
  <files>
    package.json
    knip.json
  </files>
  <action>
    1. Install knip as a dev dependency:
       ```bash
       npm install --save-dev knip
       ```

    2. Create `knip.json` at project root with this exact configuration:
       ```json
       {
         "$schema": "https://unpkg.com/knip@5/schema.json",
         "entry": [
           "src/index.ts",
           "src/api/start.ts",
           "src/cli/bin/tasks.ts",
           "src/mcp/index.ts",
           "src/db/migrate.ts"
         ],
         "project": ["src/**/*.ts"],
         "ignoreDependencies": [
           "pino-pretty"
         ]
       }
       ```

       Why `pino-pretty` is excluded: It is a runtime optional dependency loaded by pino/Fastify when `NODE_ENV !== 'production'` as a log pretty-printer. It is not imported in source code — it is convention-loaded by pino. Static analysis cannot detect this usage.

    3. Add the `lint:deps` script to package.json scripts:
       ```json
       "lint:deps": "knip --dependencies"
       ```
       Use the `--dependencies` flag to scope checks to unused dependencies only (not exports/files), which avoids false positives from incomplete entry point configuration.

    4. Run `npm run lint:deps` locally to verify it passes (exit 0) or only reports `pino-pretty` before the exclusion is applied. After the exclusion is in `knip.json`, it MUST exit 0.

    If knip reports other unexpected unused dependencies, investigate each one:
    - If truly unused: remove the dependency (npm uninstall)
    - If false positive (convention-loaded, implicit): add to `ignoreDependencies` with a comment explaining why
  </action>
  <verify>
    Run `npm run lint:deps` and confirm exit code 0 (no issues found). Run `npx knip --dependencies` directly to double-check.
  </verify>
  <done>
    `knip.json` exists with correct entry points and `pino-pretty` exclusion. `npm run lint:deps` exits 0. No unused dependencies reported.
  </done>
</task>

<task type="auto">
  <name>Task 2: Create GitHub Actions CI workflow with test and dependency check jobs</name>
  <files>
    .github/workflows/ci.yml
  </files>
  <action>
    1. Create `.github/workflows/` directory structure.

    2. Create `.github/workflows/ci.yml` with two separate jobs:

       ```yaml
       name: CI

       on:
         push:
           branches: [main]
         pull_request:
           branches: [main]

       jobs:
         test:
           runs-on: ubuntu-latest
           steps:
             - uses: actions/checkout@v4
             - uses: actions/setup-node@v4
               with:
                 node-version: '22'
                 cache: 'npm'
             - run: npm ci
             - run: npm test

         deps:
           runs-on: ubuntu-latest
           steps:
             - uses: actions/checkout@v4
             - uses: actions/setup-node@v4
               with:
                 node-version: '22'
                 cache: 'npm'
             - run: npm ci
             - name: Check unused dependencies
               run: npx knip --dependencies
       ```

       Two separate jobs (`test` and `deps`) run in parallel for faster CI. The `deps` job uses `npx knip --dependencies` directly (not `npm run lint:deps`) to be explicit about what runs in CI. knip exits with code 1 when issues are found, which fails the CI job.

    3. Verify the YAML is valid (no tabs, correct indentation).
  </action>
  <verify>
    Confirm `.github/workflows/ci.yml` exists with valid YAML structure. Confirm it has `test` and `deps` jobs. Confirm `deps` job runs `knip --dependencies`.
  </verify>
  <done>
    GitHub Actions CI workflow exists at `.github/workflows/ci.yml`. CI runs tests and unused dependency detection. CI will fail if unused dependencies are detected (knip exit code 1).
  </done>
</task>
