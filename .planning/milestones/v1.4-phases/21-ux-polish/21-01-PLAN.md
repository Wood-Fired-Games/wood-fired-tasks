    2. Read `@clack/prompts` API to understand spinner usage. The key import is `import { spinner } from '@clack/prompts';`. The spinner API is:
       - `const s = spinner();`
       - `s.start('message')` — start with a message
       - `s.stop('done message')` — stop with completion message

    3. Create `src/cli/output/spinner.ts` with:

       ```typescript
       import { spinner } from '@clack/prompts';
       import { isJsonMode } from './formatters.js';

       /**
        * Determine if the spinner should be shown.
        * Suppressed in JSON mode and when stdout is not a TTY (piped output).
        */
       function shouldShowSpinner(): boolean {
         if (isJsonMode()) return false;
         if (!process.stdout.isTTY) return false;
         return true;
       }

       /**
        * Wrap an async operation with a spinner that appears after a delay.
        *
        * @param message - Description shown while spinning (e.g., "Fetching tasks...")
        * @param fn - The async operation to execute
        * @param delay - Milliseconds to wait before showing spinner (default: 500ms)
        * @returns The result of the async operation
        */
       export async function withSpinner<T>(
         message: string,
         fn: () => Promise<T>,
         delay = 500
       ): Promise<T> {
         if (!shouldShowSpinner()) {
           return fn();
         }

         const s = spinner();
         let started = false;
         const timer = setTimeout(() => {
           started = true;
           s.start(message);
         }, delay);

         try {
           const result = await fn();
           clearTimeout(timer);
           if (started) {
             s.stop();
           }
           return result;
         } catch (error) {
           clearTimeout(timer);
           if (started) {
             s.stop();
           }
           throw error;
         }
       }
       ```

       Design decisions:
       - Default delay is 500ms (not 2 seconds) — fast enough to feel responsive but avoids flashing spinners on instant operations. The CONTEXT.md says 2 seconds is "guidance, adjust if it feels wrong" — 500ms prevents long perceived gaps.
       - Uses setTimeout to delay spinner start, clears it if operation completes before delay
       - Stop message intentionally empty — the command handler displays its own success/error message
       - shouldShowSpinner exported for testing

       Add a new helper at the bottom of the file:

       ```typescript
       import { withSpinner } from '../output/spinner.js';

       /**
        * Wrap an API call with a spinner.
        * The spinner description should match the operation (e.g., "Fetching tasks...").
        */
       export async function withApiSpinner<T>(
         message: string,
         fn: () => Promise<T>
       ): Promise<T> {
         return withSpinner(message, fn);
       }
       ```

       Note: Individual command files will use `withApiSpinner` to wrap their API calls. This is preferred over making `apiRequest` spinner-aware because:

       - The spinner is a presentation concern, not a request concern
