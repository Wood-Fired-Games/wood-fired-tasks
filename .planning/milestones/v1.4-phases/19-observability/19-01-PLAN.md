**stats.ts** — Create `src/cli/commands/stats.ts` following the `backup.ts` pattern.

The stats command runs three SQL queries against the DB (readonly mode):

1. **Task counts by status:**
```sql
SELECT status, COUNT(*) as count FROM tasks GROUP BY status ORDER BY status
```

2. **Recent activity (last 24h):**
```sql
SELECT COUNT(*) as created FROM tasks WHERE created_at >= datetime('now', '-24 hours')
```
Also query:
```sql
SELECT COUNT(*) as updated FROM tasks WHERE updated_at >= datetime('now', '-24 hours')
```

3. **Agent productivity (last 7 days):**
```sql
SELECT assignee,
       COUNT(*) as task_count,
       SUM(CASE WHEN status = 'done' THEN 1 ELSE 0 END) as completed,
       SUM(CASE WHEN status = 'in_progress' THEN 1 ELSE 0 END) as in_progress
FROM tasks
WHERE assignee IS NOT NULL
  AND updated_at >= datetime('now', '-7 days')
GROUP BY assignee
ORDER BY task_count DESC
```

Normal output format:
```
Task Counts by Status:
  open          12
  in_progress    5
  done          23
  closed         8
  backlogged     3
  Total         51

Recent Activity (24h):
  Created:  3
  Updated:  7

Agent Productivity (7 days):
  claude-1    8 tasks (5 done, 2 in progress)
  claude-2    4 tasks (3 done, 1 in progress)
```

If no tasks exist, show "No tasks found." Right-align counts using string padding. Use chalk.bold for section headers. JSON mode outputs `{ statusCounts, recentActivity: { created, updated }, agentProductivity }`.
