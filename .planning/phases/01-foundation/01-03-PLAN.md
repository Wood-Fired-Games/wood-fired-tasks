**src/services/errors.ts** -- Custom error classes:
- `ValidationError extends Error`: takes `field errors` object (Record<string, string[]>) in constructor, stores as `.fieldErrors` property. Set `this.name = 'ValidationError'`.
- `BusinessError extends Error`: takes message string. Set `this.name = 'BusinessError'`.
- `NotFoundError extends Error`: takes `entity: string` and `id: number | string` in constructor, generates message `${entity} with id ${id} not found`. Set `this.name = 'NotFoundError'`.
- All three should extend Error properly (set prototype for instanceof to work in TypeScript).

**src/schemas/task.schema.ts** -- Zod validation schemas:
- Import `z` from 'zod' and `TASK_STATUSES`, `TASK_PRIORITIES` from types.
- `CreateTaskSchema = z.object({...})`:
  - `title`: z.string().min(1, 'Title is required').max(255, 'Title must be 255 characters or less')
  - `description`: z.string().max(5000).optional().nullable()
  - `priority`: z.enum(TASK_PRIORITIES).default('medium')
  - `project_id`: z.number().int().positive('Project ID must be a positive integer')
  - `assignee`: z.string().max(100).optional().nullable()
  - `created_by`: z.string().min(1, 'Created by is required').max(100)
  - `due_date`: z.string().datetime({ message: 'Due date must be ISO8601 format' }).optional().nullable()
  - `tags`: z.array(z.string().min(1).max(50)).max(20).optional().default([])
  - Do NOT include `status` in CreateTaskSchema -- new tasks always start as 'open' (enforced by service)
- `UpdateTaskSchema = z.object({...}).partial()`:
  - `title`: z.string().min(1).max(255)
  - `description`: z.string().max(5000).nullable()
  - `status`: z.enum(TASK_STATUSES)  -- validation of transition happens in service, not schema
  - `priority`: z.enum(TASK_PRIORITIES)
  - `assignee`: z.string().max(100).nullable()
  - `due_date`: z.string().datetime().nullable()
  - `tags`: z.array(z.string().min(1).max(50)).max(20)
  - Use `.partial()` so all fields are optional (only update what's provided)
- `CreateProjectSchema = z.object({...})`:
  - `name`: z.string().min(1, 'Name is required').max(100, 'Name must be 100 characters or less')
  - `description`: z.string().max(1000).optional().nullable()
- `TaskFiltersSchema = z.object({...}).partial()`:
  - `project_id`: z.number().int().positive()
  - `status`: z.enum(TASK_STATUSES)
  - `assignee`: z.string()
  - `tags`: z.array(z.string())
  - `due_before`: z.string().datetime()
  - `due_after`: z.string().datetime()
  - `search`: z.string().min(1).max(200)
  - All `.partial()` so every filter is optional
- Export inferred types: `CreateTaskInput`, `UpdateTaskInput`, `CreateProjectInput`, `TaskFiltersInput` using `z.infer<typeof Schema>`.

**src/services/project.service.ts** -- Project business logic:
- Constructor takes `projectRepo: IProjectRepository`
- `createProject(input: unknown): Project`:
  1. Validate with CreateProjectSchema.safeParse(input)
  2. If invalid, throw ValidationError with formatted field errors
  3. Check if project name already exists via projectRepo.findByName -- if so, throw BusinessError('Project with this name already exists')
  4. Call projectRepo.create(validated data)
  5. Return created project
- `getProject(id: number): Project`:
  1. Call projectRepo.findById(id)
  2. If null, throw NotFoundError('Project', id)
  3. Return project
- `listProjects(): Project[]`:
  1. Return projectRepo.findAll()
- `updateProject(id: number, input: unknown): Project`:
  1. Validate input against a partial CreateProjectSchema
  2. Verify project exists (throw NotFoundError if not)
  3. If name is being changed, check uniqueness
  4. Call projectRepo.update(id, validated data)
  5. Return updated project
- `deleteProject(id: number): void`:
  1. Verify project exists (throw NotFoundError if not)
  2. Call projectRepo.delete(id)

**src/services/task.service.ts** -- Task business logic:
- Constructor takes `taskRepo: ITaskRepository` and `projectRepo: IProjectRepository` (need project repo to verify project exists when creating tasks).

**createTask(input: unknown): Task & { tags: string[] }**
  1. Validate with CreateTaskSchema.safeParse(input)
  2. If invalid, throw ValidationError with field errors
  3. Verify project exists: call projectRepo.findById(validated.project_id). If null, throw BusinessError(`Project with id ${project_id} does not exist`)
  4. Call taskRepo.create({ ...validated, status: 'open' }, validated.tags) -- always force status to 'open'
  5. Return created task

**getTask(id: number): Task & { tags: string[] }**
  1. Call taskRepo.findById(id)
  2. If null, throw NotFoundError('Task', id)
  3. Return task

**listTasks(filters?: unknown): Array<Task & { tags: string[] }>**
  1. If filters provided, validate with TaskFiltersSchema.safeParse(filters)
  2. If invalid filters, throw ValidationError
  3. Call taskRepo.findByFilters(validated filters or {})
  4. Return results

**updateTask(id: number, input: unknown): Task & { tags: string[] }**
  1. Validate with UpdateTaskSchema.safeParse(input)
  2. If invalid, throw ValidationError
  3. Fetch existing task: taskRepo.findById(id). If null, throw NotFoundError
  4. If status is being changed, validate transition:
     - Import VALID_STATUS_TRANSITIONS from types
     - Get current status from existing task
     - Check if new status is in VALID_STATUS_TRANSITIONS[currentStatus]
     - If not, throw BusinessError with message listing valid transitions: `Invalid status transition from '${current}' to '${new}'. Valid transitions: ${validTargets.join(', ')}`
  5. Call taskRepo.update(id, validated updates)
  6. Return updated task

**deleteTask(id: number): void**
  1. Fetch existing task to verify it exists. If null, throw NotFoundError
  2. Call taskRepo.delete(id)

**countTasks(filters?: unknown): number**
  1. If filters, validate with TaskFiltersSchema
  2. Return taskRepo.count(validated filters or undefined)

**searchTasks(query: string): Array<Task & { tags: string[] }>**
  - Convenience method: calls listTasks({ search: query })
