**src/repositories/task.repository.ts** -- Implement ITaskRepository:
- Constructor takes `db: Database.Database`
- Prepare reusable statements: insert task, findById, delete, find tags by task_id, insert tag, delete tags by task_id.

**create(dto, tags?):**
- Wrap in `db.transaction()`:
  1. INSERT into tasks with all DTO fields, `created_at` and `updated_at` set to `new Date().toISOString()`
  2. Get lastInsertRowid
  3. If tags provided and non-empty, INSERT each into task_tags (task_id, tag)
  4. Return findById result (which includes tags)

**findById(id):**
- SELECT task by id
- If null, return null
- SELECT all tags for this task_id from task_tags
- Return task object with `tags: string[]` property added

**findAll():**
- SELECT all tasks ORDER BY created_at DESC
- For each task, load its tags (use a single query with GROUP_CONCAT or a separate tags query)
- Prefer a LEFT JOIN approach: `SELECT t.*, GROUP_CONCAT(tt.tag) as tags_csv FROM tasks t LEFT JOIN task_tags tt ON tt.task_id = t.id GROUP BY t.id ORDER BY t.created_at DESC`
- Parse tags_csv into string array (split on comma, handle null as empty array)

**update(id, updates):**
- Wrap in `db.transaction()`:
  1. Build dynamic UPDATE SET clause from non-undefined fields in updates (exclude `tags` from the SQL update)
  2. Always add `updated_at = datetime('now')` to the SET clause
  3. If `updates.tags` is provided (even empty array), delete all existing tags for this task_id, then INSERT new tags
  4. Return findById result
- IMPORTANT: Use parameterized queries for dynamic updates. Build the SET clause as `field = @field` pairs, not string interpolation of values.

**delete(id):**
- DELETE FROM tasks WHERE id = ? (CASCADE handles task_tags)

**findByFilters(filters):**
- Build dynamic WHERE clause from provided filters (see research Pattern: Complex Filter Query):
  - `project_id` -> `t.project_id = @project_id`
  - `status` -> `t.status = @status`
  - `assignee` -> `t.assignee = @assignee`
  - `due_before` -> `t.due_date <= @due_before`
  - `due_after` -> `t.due_date >= @due_after`
  - `tags` -> `EXISTS (SELECT 1 FROM task_tags tt WHERE tt.task_id = t.id AND tt.tag IN (...))`  with parameterized tag values
  - `search` -> `t.id IN (SELECT rowid FROM tasks_fts WHERE tasks_fts MATCH @search)` -- use FTS5 MATCH
- JOIN with task_tags via GROUP_CONCAT for tags in results
- ORDER BY t.created_at DESC
- Return array of Task & { tags: string[] }
- If no filters provided, return all tasks (same as findAll)

**count(filters?):**
- Similar to findByFilters but SELECT COUNT(DISTINCT t.id) instead of rows
