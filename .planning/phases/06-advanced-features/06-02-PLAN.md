**Migration (003-comments-and-estimates.ts):**
Follow the exact pattern from 001/002 (up/down functions, db.transaction wrapper).
1. Create task_comments table:
   ```sql
   CREATE TABLE task_comments (
     id INTEGER PRIMARY KEY AUTOINCREMENT,
     task_id INTEGER NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
     author TEXT NOT NULL,
     content TEXT NOT NULL,
     created_at TEXT NOT NULL DEFAULT (datetime('now')),
     updated_at TEXT
   )
   ```
2. `CREATE INDEX idx_comments_task_created ON task_comments(task_id, created_at)` -- composite index for chronological retrieval
3. `ALTER TABLE tasks ADD COLUMN estimated_minutes INTEGER` -- nullable integer for time estimates

The `down` function: `ALTER TABLE tasks DROP COLUMN estimated_minutes`, then `DROP TABLE task_comments`.

**REST API -- Dependency Routes (src/api/routes/dependencies/index.ts):**
Create as FastifyPluginAsyncZod (same pattern as tasks/index.ts). Routes are nested under tasks:
- `POST /api/v1/tasks/:id/dependencies` -- Add dependency. Body: `{ blocks_task_id: number }`. The `:id` param is the task_id (the task that blocks). Calls dependencyService.addDependency({ task_id: params.id, blocks_task_id: body.blocks_task_id }). Returns 201 with dependency object.
- `GET /api/v1/tasks/:id/dependencies` -- Get dependencies for a task. Returns `{ blocks: Dependency[], blocked_by: Dependency[] }` where blocks = tasks this task blocks, blocked_by = tasks that block this task.
- `DELETE /api/v1/tasks/:id/dependencies/:blocksTaskId` -- Remove dependency. Calls dependencyService.removeDependency(params.id, params.blocksTaskId). Returns 204.

**Dependency Schemas (src/api/routes/dependencies/schemas.ts):**
- `DependencyResponseSchema` -- z.object matching Dependency type
- `DependencyListResponseSchema` -- z.object({ blocks: z.array(DependencyResponseSchema), blocked_by: z.array(DependencyResponseSchema) })
- `CreateDependencyBodySchema` -- z.object({ blocks_task_id: z.coerce.number().int().positive() })

**REST API -- Comment Routes (src/api/routes/comments/index.ts):**
Nested under tasks:
- `POST /api/v1/tasks/:id/comments` -- Add comment. Body: `{ author: string, content: string }`. Calls commentService.addComment({ task_id: params.id, ...body }). Returns 201 with comment object.
- `GET /api/v1/tasks/:id/comments` -- Get comments for a task. Calls commentService.getComments(params.id). Returns array of comments (already chronological).
- `DELETE /api/v1/tasks/:id/comments/:commentId` -- Delete comment. Calls commentService.deleteComment(params.commentId). Returns 204.
