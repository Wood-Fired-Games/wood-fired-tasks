**Migration (002-task-hierarchy-and-dependencies.ts):**
Follow the exact pattern from 001-initial-schema.ts (up/down functions, db.transaction wrapper). In the `up` function:
1. `ALTER TABLE tasks ADD COLUMN parent_task_id INTEGER REFERENCES tasks(id) ON DELETE CASCADE` -- self-referencing FK for subtask hierarchy
2. `CREATE INDEX idx_tasks_parent_id ON tasks(parent_task_id)` -- index for child lookups
3. Create `task_dependencies` table:
   ```sql
   CREATE TABLE task_dependencies (
     id INTEGER PRIMARY KEY AUTOINCREMENT,
     task_id INTEGER NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
     blocks_task_id INTEGER NOT NULL REFERENCES tasks(id) ON DELETE CASCADE,
     created_at TEXT NOT NULL DEFAULT (datetime('now')),
     UNIQUE(task_id, blocks_task_id),
     CHECK(task_id != blocks_task_id)
   )
   ```
4. `CREATE INDEX idx_dependencies_task ON task_dependencies(task_id)`
5. `CREATE INDEX idx_dependencies_blocked ON task_dependencies(blocks_task_id)`

The `down` function: DROP TABLE task_dependencies, then remove parent_task_id column. Note: SQLite does not support DROP COLUMN before 3.35.0. Since we target recent SQLite, use `ALTER TABLE tasks DROP COLUMN parent_task_id`. If that proves problematic, use the table-rebuild pattern (CREATE new table, copy data, drop old, rename). Check the SQLite version available at test time -- better-sqlite3 12.x bundles SQLite 3.46+, so DROP COLUMN is supported.

**DependencyService (src/services/dependency.service.ts):**
Follow the exact pattern from task.service.ts (constructor injection, Zod validation, custom errors):
- Constructor takes `IDependencyRepository` and `ITaskRepository` (need task repo to verify both tasks exist)
- `addDependency(input: unknown): Dependency`:
  1. Validate with CreateDependencySchema.safeParse (throw ValidationError on failure, using same fieldErrors pattern as TaskService)
  2. Verify both task_id and blocks_task_id exist via taskRepo.findById (throw NotFoundError if either missing)
  3. Load all existing deps via dependencyRepo.findAll()
  4. Create CycleDetector with existing deps
  5. Call wouldCreateCycle(task_id, blocks_task_id) -- if true, throw BusinessError with message "Cannot add dependency: Task {task_id} blocking Task {blocks_task_id} would create a circular dependency"
  6. Call dependencyRepo.create() and return result
- `removeDependency(taskId: number, blocksTaskId: number): void` -- delete via repo, throw NotFoundError if dependency didn't exist (delete returns false)
- `getBlockedBy(taskId: number): Dependency[]` -- tasks blocked by this task (dependencyRepo.findByTaskId)
- `getBlockers(taskId: number): Dependency[]` -- tasks blocking this task (dependencyRepo.findBlockingTask)
