YAML frontmatter:
- name: project-status
- description: "Shows project overview with task counts grouped by status and completion percentage. Use when user asks about project status, progress, overview, dashboard, or summary."
- disable-model-invocation: false

Markdown body (aggregation workflow — this is the most complex skill):

1. Call `wood-fired-bugs:list_projects` with empty parameters to get all projects.

2. For each project returned:
   a. Call `wood-fired-bugs:list_tasks` with filter: { project_id: project.id }
   b. Group tasks by status: open, in_progress, done, closed, blocked
   c. Calculate completion percentage: (done + closed) / total * 100

3. Format output for each project:
   ```
   ## <Project Name>
   Total: <count> tasks | Completion: <percentage>%
   - Open: <count>
   - In Progress: <count>
   - Done: <count>
   - Blocked: <count>
   - Closed: <count>
   ```

4. Add highlights section at the end:
   - Flag projects with blocked tasks (any blocked count > 0) as needing attention
   - Flag projects with high completion (>80%) as nearly done
   - Show overall totals across all projects

5. If no projects exist, display message suggesting /tasks:create-task to get started.

6. If $ARGUMENTS contains a project name or ID, filter to show only that project's details instead of all projects.

Keep under 80 lines total. Use imperative form throughout.
Do NOT use emojis in the skill file body. Use text labels like "ATTENTION:" instead.
