**Add dependency auto-unblock logic to handleStatusChanged:**

After parent auto-complete check, add:
1. If event data status changed to 'done':
   - Find all tasks that this task blocks: `dependencyRepo.findByTaskId(taskId)` returns Dependency[] where task_id = completedTaskId. Each dependency.blocks_task_id is a task that was blocked by this now-completed task.
   - For each blocked task:
     a. Get ALL blockers of that task: `dependencyRepo.findBlockingTask(blockedTaskId)` returns Dependency[] where blocks_task_id = blockedTaskId
     b. For each blocker, check if the blocking task (dependency.task_id) is done: `taskRepo.findById(dependency.task_id)?.status === 'done'`
     c. If ALL blockers are done AND the blocked task status is 'blocked':
        - Call `taskService.updateTask(blockedTaskId, { status: 'open' }, 'workflow')`
     d. If not all blockers are done, skip (task remains blocked)

**Important:** The dependency auto-unblock also participates in cascade depth tracking. Unblocking a task emits task.status_changed, which could trigger further workflow processing (e.g., if the unblocked task's parent has all children done).
