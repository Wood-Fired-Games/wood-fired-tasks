<feature>
  <name>EventBus with TypeScript Generics</name>
  <files>src/events/event-bus.ts, src/events/__tests__/event-bus.test.ts, src/events/types.ts</files>
  <behavior>
    EventBus provides type-safe pub/sub for domain events:

    Test cases:
    1. emit('task.created', payload) → all subscribers receive payload
    2. subscribe(handler) → handler called with correct typed payload
    3. unsubscribe() → handler no longer receives events
    4. emit with no subscribers → no errors thrown
    5. multiple subscribers → all receive same event
    6. subscriber throws error → other subscribers still execute

    Event types to support:
    - task.created, task.updated, task.deleted
    - task.claimed, task.status_changed
    - project.created, project.updated, project.deleted

    Payload structure:
    - eventType: string (e.g., "task.created")
    - timestamp: ISO 8601 string
    - data: entity snapshot (full Task or Project object)
    - metadata: { source: 'user' | 'workflow', actor?: string }
  </behavior>
  <implementation>
    After tests pass:

    1. Create src/events/types.ts:
       - TaskEventType union ('task.created' | 'task.updated' | ...)
       - ProjectEventType union
       - EventPayload<T> generic interface { eventType, timestamp, data: T, metadata }
       - TaskEvent = EventPayload<Task & { tags: string[] }>
       - ProjectEvent = EventPayload<Project>

    2. Create EventBus extending EventEmitter:
       - Generic class EventBus<Events extends Record<string, unknown>>
       - emit<K extends keyof Events>(event: K, payload: Events[K]): void
       - subscribe<K extends keyof Events>(event: K, handler: (payload: Events[K]) => void): () => void
       - unsubscribe handled by returned cleanup function
       - Use super.on/super.emit internally with type safety wrapper

    3. Export singleton instance for application use:
       - type AppEvents = { 'task.created': TaskEvent, 'task.updated': TaskEvent, ... }
       - export const eventBus = new EventBus<AppEvents>()

    Research finding: EventEmitter has typed generics support since @types/node July 2024, so no wrapper needed - just extend with typed interface.

    Pitfall addressed: Subscriber errors must not crash EventBus - wrap handler calls in try/catch.
  </implementation>
</feature>
