<task type="auto">
  <name>Task 1: Fix stdout pollution and audit MCP server logging</name>
  <files>src/db/migrate.ts</files>
  <action>
Fix the Umzug logger stdout pollution bug in `src/db/migrate.ts` line 69.

**The bug:** `logger: console` passes the global console object to Umzug. Umzug calls `logger.info()` for migration events (e.g., "migrating 001-initial.ts", "migrated 001-initial.ts"). In Node.js, `console.info()` writes to **stdout**, which corrupts the MCP JSON-RPC stream when the MCP server starts up and runs migrations.

**The fix:** Replace `logger: console` with a custom logger that routes all output to stderr:

```typescript
logger: {
  info: (msg: Record<string, unknown>) => console.error('[migration]', msg),
  warn: (msg: Record<string, unknown>) => console.error('[migration:warn]', msg),
  error: (msg: Record<string, unknown>) => console.error('[migration:error]', msg),
  debug: (msg: Record<string, unknown>) => console.error('[migration:debug]', msg),
},
```

This preserves migration logging visibility (via stderr) while keeping stdout clean for JSON-RPC.

**Also verify (no changes expected):**
- `src/mcp/index.ts` — already uses console.error only (confirmed)
- `src/mcp/errors.ts` — already uses console.error only (confirmed)
- `src/mcp/tools/health-tools.ts` — already uses console.error only (confirmed)
- `src/mcp/server.ts` — has no console output (confirmed)
- All other `src/mcp/tools/*.ts` — no console output (confirmed by grep)
- `src/index.ts` line 67 `console.log` — safe, guarded by `import.meta.url` check so only runs when index.ts is the direct entry point, not when imported by MCP server

**Do NOT** change any console.log calls in `src/cli/` — those are CLI output, not MCP server.
