  <name>Install @clack/prompts and create prompt utilities</name>
  <files>
    package.json
    src/cli/prompts/interactive.ts
  </files>
  <action>
    Install @clack/prompts dependency:
    npm install @clack/prompts@^1.0.1

    Create src/cli/prompts/interactive.ts with:

    1. shouldPrompt() function:
       - Returns false if --no-input flag set OR !process.stdin.isTTY
       - Returns true otherwise
       - Access --no-input via process.argv check (consistent with formatters approach)

    2. promptForMissing<T>(field: string, value: T | undefined, options?: {defaultValue?: T, validate?: (v: string) => boolean}): Promise<T>:
       - If value is defined, return it immediately
       - If !shouldPrompt(), throw error: "Missing required field: {field}. Use --{field} or remove --no-input"
       - Otherwise, use @clack/prompts text() to prompt user
       - Return validated input

    3. confirmAction(message: string, defaultValue = false): Promise<boolean>:
       - If --force flag set, return true immediately (skip confirmation)
       - If !process.stdin.isTTY, throw error: "Confirmation required. Use --force or run in interactive terminal"
       - Otherwise, use @clack/prompts confirm() to show Y/N prompt
       - Return user choice

    Export all three functions with TypeScript types.

    Implementation notes from research:
    - @clack/prompts handles cancellation (Ctrl+C) automatically
    - Use text() for string input, confirm() for Y/N
    - TTY check prevents hanging in CI/scripts
    - --no-input and --force flags make CLI script-friendly
  </action>
  <verify>
    grep -q '@clack/prompts' package.json
    npx tsc --noEmit

    grep -q "export.*promptForMissing" src/cli/prompts/interactive.ts
    grep -q "export.*confirmAction" src/cli/prompts/interactive.ts
    grep -q "process.stdin.isTTY" src/cli/prompts/interactive.ts
  </verify>
