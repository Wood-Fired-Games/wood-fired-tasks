Create `src/slack/commands/tasks-command.ts` with:

1. **Services interface:**
```typescript
import type { App } from '@slack/bolt';
import type { RespondFn, SlashCommand } from '@slack/bolt';
import type { KnownBlock, SectionBlock } from '@slack/types';
import type { TaskService } from '../../services/task.service.js';
import type { ProjectService } from '../../services/project.service.js';
import type { DependencyService } from '../../services/dependency.service.js';
import type { CommentService } from '../../services/comment.service.js';
import type { UserIdentityCache } from '../user-identity.js';
import { NotFoundError, ValidationError, BusinessError } from '../../services/errors.js';

export interface Services {
  taskService: TaskService;
  projectService: ProjectService;
  dependencyService: DependencyService;
  commentService: CommentService;
}
```

2. **parseArgs utility** — splits args array into `{ positionals: string[], flags: Record<string, string> }`. Iterates args: if starts with `--`, take next arg as value and skip it; otherwise push to positionals.

3. **respondBlocks helper** — calls `respond({ response_type: 'ephemeral', blocks, text: fallbackText })`.

4. **respondError helper** — calls `respond({ response_type: 'ephemeral', blocks: [SectionBlock with mrkdwn `:x: ${message}\n${hint}`], text: message })`. Optional hint parameter.

5. **formatServiceError function** — checks `instanceof NotFoundError`, `instanceof ValidationError` (format fieldErrors as `field: msg1, msg2`), `instanceof BusinessError`, `instanceof Error`, fallback `'Unexpected error'`.

6. **HELP_BLOCKS constant** — a KnownBlock[] with:
   - HeaderBlock: "Tasks -- Available Commands"
   - SectionBlock (mrkdwn): Task commands (list, show, create, update, claim, delete)
   - DividerBlock
   - SectionBlock (mrkdwn): Project commands (project-list, project-show, project-create, project-update, project-delete)
   - DividerBlock
   - SectionBlock (mrkdwn): Dependency, Comment, Subtask commands (dep-add, dep-list, dep-remove, comment-add, comment-list, comment-delete, subtask-create, subtask-list)
   Each with backtick-wrapped usage examples per the research Pattern 8.

7. **registerTasksCommand function** — exported. Takes `(app: App, services: Services, identityCache: UserIdentityCache)`. Calls `app.command('/tasks', async ({ ack, respond, command }) => { ... })`. Inside:
   - `await ack()` — FIRST statement, no exceptions
   - Parse: `const text = command.text.trim(); const [subcommand, ...args] = text ? text.split(/\s+/) : [''];`
   - Handle empty string edge case: `''.split(/\s+/)` returns `['']`, so check `if (!subcommand || subcommand === '')` to redirect to help
   - switch on subcommand with cases for all 26 subcommands (task: list, show, create, update, delete, claim; project: project-create, project-list, project-show, project-update, project-delete; dependency: dep-add, dep-list, dep-remove; comment: comment-add, comment-list, comment-delete; subtask: subtask-create, subtask-list; operational: health, backup, doctor, stats, db-check, completions). For now, all cases except 'help' and the default/empty case call `await respondError(respond, 'Not yet implemented: \`' + subcommand + '\`')` — Plans 02 and 03 will fill these in.
   - 'help' case: `await respondBlocks(respond, HELP_BLOCKS, 'Tasks -- Available Commands')`
   - default case: `await respondError(respond, 'Unknown subcommand: \`' + subcommand + '\`', 'Run \`/tasks help\` to see available subcommands.')`
   - Wrap the entire switch in try/catch: catch calls `await respondError(respond, formatServiceError(error))`
