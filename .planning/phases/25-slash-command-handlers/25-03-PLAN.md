  <name>Task 1: Implement project, dependency, comment, subtask, health, and CLI-only stub handlers</name>
  <files>src/slack/commands/tasks-command.ts</files>
  <action>
Add import for project formatters:
```typescript
import { formatProjectList, formatProjectDetail } from '../formatters/project-formatter.js';
```

Implement and wire these handler functions into the switch, replacing remaining stubs:

**Project handlers:**

1. **handleProjectList(respond, services):**
   - Call `services.projectService.listProjects()`
   - Format with `formatProjectList(projects)` and respond

2. **handleProjectShow(respond, services, args):**
   - Parse id from args[0]. If NaN, respondError with usage.
   - Call `services.projectService.getProject(id)`
   - Format with `formatProjectDetail(project)` and respond

3. **handleProjectCreate(respond, services, args):**
   - Parse args with parseArgs. Join positionals as name. Flag: `--description`.
   - If no name, respondError with usage.
   - Call `services.projectService.createProject({ name, description: flags.description || null })`
   - Format with `formatProjectDetail(project)` and respond

4. **handleProjectUpdate(respond, services, args):**
   - First arg is project ID. Parse remaining with parseArgs for `--name`, `--description`.
   - Build updates from present flags. If no flags, respondError.
   - Call `services.projectService.updateProject(id, updates)` and format with `formatProjectDetail`

5. **handleProjectDelete(respond, services, args):**
   - Parse id. Call `services.projectService.deleteProject(id)`
   - Respond with confirmation: ":white_check_mark: Project #<id> deleted."

**Dependency handlers:**

6. **handleDepAdd(respond, services, args):**
   - Parse two positional IDs: `args[0]` (task_id) and `args[1]` (blocks_task_id). Both required, both must be numeric.
   - Call `services.dependencyService.addDependency({ task_id, blocks_task_id })`
   - Respond with confirmation: ":white_check_mark: Dependency added: Task #X blocks Task #Y"

7. **handleDepList(respond, services, args):**
   - Parse id from args[0].
   - Call `services.dependencyService.getBlockedBy(id)` and `services.dependencyService.getBlockers(id)`
   - Format as SectionBlock with mrkdwn: "*Dependencies for Task #<id>*\n*Blocks:* #X, #Y (or _none_)\n*Blocked by:* #A, #B (or _none_)"
   - Respond with blocks

8. **handleDepRemove(respond, services, args):**
   - Parse two positional IDs.
   - Call `services.dependencyService.removeDependency(taskId, blocksTaskId)`
   - Respond with confirmation: ":white_check_mark: Dependency removed: Task #X no longer blocks Task #Y"

**Comment handlers:**

9. **handleCommentAdd(respond, services, identityCache, command, args):**
   - First arg is task_id. Everything after is content: `args.slice(1).join(' ')`. (Pitfall 7 from research: must rejoin for multi-word content.)
   - If no task_id or no content, respondError.
   - Resolve display name: `const author = await identityCache.resolve(command.user_id)` (consistent UIDENT-03 pattern)
   - Call `services.commentService.addComment({ task_id: id, author, content })`
   - Respond with confirmation: ":white_check_mark: Comment added to Task #<id>"

10. **handleCommentList(respond, services, args):**
    - Parse task_id from args[0].
    - Call `services.commentService.getComments(id)`
    - Format as blocks: HeaderBlock "Comments for Task #<id>", then SectionBlock per comment with "*<author>* (<created_at>)\n<content>". If empty, respond with "_No comments._"

11. **handleCommentDelete(respond, services, args):**
    - Parse two positional args: task_id and comment_id.
    - Call `services.commentService.deleteComment(commentId)` (service validates existence)
    - Respond with confirmation: ":white_check_mark: Comment #<comment-id> deleted."

**Subtask handlers:**

12. **handleSubtaskCreate(respond, services, identityCache, command, args):**
    - First arg is parent_task_id. Remaining positionals are title (joined with spaces). Flag: `--project` (required).
    - If no title, respondError. If no --project, respondError.
    - Resolve display name for created_by.
    - Call `services.taskService.createTask({ title, project_id, parent_task_id: parentId, priority: flags.priority || 'medium', created_by: displayName })`
    - Format with `formatTaskDetail(task)` and respond

13. **handleSubtaskList(respond, services, args):**
    - Parse parent_task_id from args[0].
    - Call `services.taskService.getSubtasks(parentId)`
    - Format with `formatTaskList(subtasks)` and respond

**Health handler:**

14. **handleHealth(respond, services):**
    - Call `services.taskService.countTasks()` in a try/catch
    - On success: respond with SectionBlock ":white_check_mark: Service is healthy. ${count} tasks in database."
    - On error: respond with SectionBlock ":x: Service health check failed."

**CLI-only stubs (5 commands):**

15. For `backup`, `doctor`, `stats`, `db-check`, `completions`: respond with SectionBlock:
    `:information_source: \`${subcommand}\` is only available via the CLI.\nRun: \`tasks ${subcommand}\``

Wire all handlers into the switch statement. Each case calls the appropriate handler with the correct arguments (respond, services, args, identityCache, command as needed).
