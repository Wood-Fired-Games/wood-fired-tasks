**Function: registerProjectTools(server: McpServer, projectService: ProjectService): void**

**Tool 1: create_project**
- Description: "Create a new project"
- inputSchema: CreateProjectSchema (name, description optional)
- Handler: Call projectService.createProject(args), wrap in try/catch with convertToMcpError
- Return: text content "Project created: {name} (ID: {id})" + structuredContent with project object
- Use double type assertion for structuredContent: `project as unknown as { [x: string]: unknown }`

**Tool 2: get_project**
- Description: "Get a project by its ID"
- inputSchema: z.object({ id: z.number().int().positive() })
- Handler: Call projectService.getProject(args.id)
- Return: text summary with name, description (if present), created_at + structuredContent
- Text format: "Project: {name}\nCreated: {created_at}\n[Description: {description}]"

**Tool 3: list_projects**
- Description: "List all projects"
- inputSchema: z.object({}) (no parameters)
- Handler: Call projectService.listProjects()
- Return: text "Found {count} project(s):" with bulleted list + structuredContent { projects }
- Text format: "- [{id}] {name}"
- Handle empty case: "No projects found." with structuredContent { projects: [] }

**Tool 4: update_project**
- Description: "Update an existing project by ID. Can update name and/or description."
- inputSchema: z.object({ id: z.number().int().positive(), updates: CreateProjectSchema.partial() })
- Handler: Call projectService.updateProject(args.id, args.updates)
- Return: text "Project {id} updated: {name}" + structuredContent with updated project

**Tool 5: delete_project**
- Description: "Delete a project by its ID"
- inputSchema: z.object({ id: z.number().int().positive() })
- Handler: Call projectService.deleteProject(args.id)
- Return: text "Project {id} deleted successfully." (no structuredContent)

**Patterns to follow:**
- All handlers are async but call synchronous service methods (better-sqlite3 is sync)
- Use console.error() for any logging (not console.log - stdout reserved for JSON-RPC)
- Match formatting style from task-tools.ts exactly
- All server.registerTool calls follow same pattern: name, config object, handler
