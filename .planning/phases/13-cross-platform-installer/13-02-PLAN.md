<task type="auto">
  <name>Task 1: Create PowerShell installer script</name>
  <files>install.ps1</files>
  <action>
Create install.ps1 at project root with these sections, mirroring the Bash installer's behavior:

**Header and safety:**
- `$ErrorActionPreference = "Stop"` — stop on any error
- Wrap entire body in try/catch/finally for rollback on failure
- Store backup path in script-scoped variable for cleanup access

**Constants:**
- `$ScriptDir` — resolved directory of the script itself (`$PSScriptRoot`)
- `$ConfigFile = "$env:USERPROFILE\.claude.json"` — Claude Code config location
- `$SkillsSource = Join-Path $ScriptDir "skills" "tasks"` — source skill files
- `$SkillsDest = Join-Path $env:USERPROFILE ".claude" "commands" "tasks"` — destination
- `$ServiceUrl` — default "http://localhost:3000", allow override via `$env:WOOD_FIRED_BUGS_URL`

**Step 1: Check prerequisites:**
- Verify `node` is accessible (`Get-Command node -ErrorAction SilentlyContinue`)
- Verify skills source directory exists at `$SkillsSource`
- Count .md files in source, print "Found N skill files"
- No jq needed (PowerShell has native JSON support)

**Step 2: Prompt for API key:**
- Accept via `-ApiKey` parameter, `$env:WOOD_FIRED_BUGS_API_KEY`, or interactive `Read-Host -MaskInput` (in that priority order)
- Validate key is non-empty
- Print masked key confirmation (first 4 chars + asterisks)

**Step 3: Copy skill files (WIN-01):**
- `New-Item -ItemType Directory -Force -Path $SkillsDest | Out-Null`
- Copy all .md files from `$SkillsSource` to `$SkillsDest` using `Copy-Item -Force`
- Only copy files that are newer or don't exist at destination (idempotent)
- Print count of files copied

**Step 4: Backup existing config (WIN-04):**
- If config file exists, create timestamped backup: `$ConfigFile.backup.yyyyMMdd_HHmmss`
- Print backup location
- If config does not exist, create file with empty JSON object `{}`

**Step 5: Merge MCP server config (WIN-02, WIN-03):**
- Read existing config with `Get-Content | ConvertFrom-Json`
- Build new server config as PSCustomObject:
  ```
  wood-fired-bugs = @{
    command = "node"
    args = @("dist/mcp/index.js")
    cwd = "$ScriptDir"
    env = @{
      WOOD_FIRED_BUGS_API_KEY = "$ApiKey"
      DB_PATH = "./data/tasks.db"
    }
  }
  ```
- If `mcpServers` property doesn't exist on config, add it as empty PSCustomObject
- Add or update `wood-fired-bugs` property on `mcpServers` using `Add-Member -Force`
- Write back with `ConvertTo-Json -Depth 10 | Set-Content $ConfigFile`
- Print confirmation with server name

**Step 6: Validate connectivity (WIN-05):**
- Use `Invoke-WebRequest` with `-TimeoutSec 5` to check `$ServiceUrl/health`
- Wrap in try/catch — failure is non-fatal warning
- If reachable: print success with "[OK]" prefix
- If not reachable: print "[WARN]" — service may not be running yet
- Print note: "Restart Claude Code to apply configuration changes"

**Step 7: Summary:**
- Print summary of what was done (same structure as Bash installer)
- Print: "Installation complete. Run /tasks: in Claude Code to get started."

**Error handling:**
- catch block: Print error message, restore backup if it exists, exit 1
- Use `Write-Host` with `-ForegroundColor` for colored output (Red for errors, Yellow for warnings, Green for success)

**Style notes:**
- Use text labels "[OK]", "[WARN]", "[ERROR]", "[INFO]" instead of emojis (accessibility)
- Use `Join-Path` for all path construction (handles backslash correctly)
- The `cwd` field uses resolved absolute path from `$ScriptDir`
- Add `-ApiKey` as a script parameter for non-interactive use
