<task type="auto">
  <name>Task 1: Create Bash installer script</name>
  <files>install.sh</files>
  <action>
Create install.sh at project root with these sections, in order:

**Header and safety:**
- Shebang: `#!/usr/bin/env bash`
- `set -euo pipefail`
- trap handler for cleanup/rollback on failure

**Constants:**
- `SCRIPT_DIR` — resolved directory of the script itself (for finding skills/tasks/ relative to it)
- `CONFIG_FILE="$HOME/.claude.json"` — Claude Code config location
- `SKILLS_SOURCE="$SCRIPT_DIR/skills/tasks"` — source skill files
- `SKILLS_DEST="$HOME/.claude/commands/tasks"` — destination for skills
- `SERVICE_URL` — default "http://localhost:3000" (allow override via env var `WOOD_FIRED_BUGS_URL`)

**Step 1: Check prerequisites:**
- Verify `jq` is installed (`command -v jq`), provide install instructions for Debian/Ubuntu, RHEL, macOS if missing
- Verify `curl` is installed
- Verify skills source directory exists at `$SCRIPT_DIR/skills/tasks/`
- Count .md files in source, print "Found N skill files"

**Step 2: Prompt for API key:**
- Accept API key via `--api-key` flag, `WOOD_FIRED_BUGS_API_KEY` env var, or interactive `read -sp` prompt (in that priority order)
- Validate key is non-empty
- Print masked key confirmation (first 4 chars + asterisks)

**Step 3: Copy skill files (LINX-01):**
- `mkdir -p "$SKILLS_DEST"`
- Copy all .md files from `$SKILLS_SOURCE/` to `$SKILLS_DEST/` using `cp -a`
- Print count of files copied

**Step 4: Backup existing config (LINX-04):**
- If `$CONFIG_FILE` exists, create timestamped backup: `${CONFIG_FILE}.backup.YYYYMMDD_HHMMSS`
- Print backup location
- If config does not exist, create empty JSON object `{}` as initial config

**Step 5: Merge MCP server config (LINX-02, LINX-03, LINX-05):**
- Build MCP server JSON fragment with jq:
  ```
  {
    "mcpServers": {
      "wood-fired-bugs": {
        "command": "node",
        "args": ["dist/mcp/index.js"],
        "cwd": "$SCRIPT_DIR",
        "env": {
          "WOOD_FIRED_BUGS_API_KEY": "$API_KEY",
          "DB_PATH": "./data/tasks.db"
        }
      }
    }
  }
  ```
- Use `jq -s '.[0] * .[1]'` for deep merge (preserves existing servers)
- Write merged result to config via temp file + mv (atomic write)
- Clean up temp files
- Print confirmation with server name

**Step 6: Validate connectivity (LINX-05):**
- Use `curl --fail --silent --connect-timeout 5 "$SERVICE_URL/health"` to check service
- If reachable: print success message with "[OK]" prefix
- If not reachable: print warning (not error) — service may not be running yet, this is non-fatal
- Print note: "Restart Claude Code to apply configuration changes"

**Step 7: Summary:**
- Print summary of what was done:
  - N skill files installed to [path]
  - MCP server configured in [config path]
  - Backup saved to [backup path] (if applicable)
  - Service status (reachable or not)
- Print: "Installation complete. Run /tasks: in Claude Code to get started."

**Style notes:**
- Use text labels like "[OK]", "[WARN]", "[ERROR]", "[INFO]" instead of emojis (accessibility, per Phase 12 decision)
- Use printf or echo for output, not fancy formatting libraries
- All paths should be quoted to handle spaces
- The `cwd` field in MCP config uses absolute path resolved from `$SCRIPT_DIR`
- Make the script executable (`chmod +x` is not needed in the script itself, but note it in output)
