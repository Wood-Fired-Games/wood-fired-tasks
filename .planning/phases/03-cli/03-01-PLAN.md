  <action>
1. Install CLI dependencies:
   ```
   npm install commander cli-table3 chalk@4 dotenv
   ```
   Note: chalk must be v4.x (not 5.x) because chalk 5 is ESM-only and has import issues with TypeScript module resolution. chalk 4.x works with both CJS and ESM via esModuleInterop.

2. Create `.env.example` at project root with:
   ```
   API_BASE_URL=http://localhost:3000
   API_KEY=your-api-key-here
   ```

3. Create `src/cli/config/env.ts`:
   - Import dotenv and call `dotenv.config()` to load from project root `.env` file. Use `import { fileURLToPath } from 'url'` and `path.dirname()` to resolve the `.env` path relative to the project root (3 levels up from `src/cli/config/`).
   - Export an `env` object with `API_BASE_URL` (string, defaults to `http://localhost:3000`) and `API_KEY` (string, required).
   - If `API_KEY` is not set, print a clear error to stderr using `console.error()` explaining the user needs to set API_KEY in `.env` or as an environment variable, then call `process.exit(1)`.
   - Validate at module load time (not lazily) so missing config fails fast.

4. Create `src/cli/api/types.ts`:
   - Define TypeScript interfaces matching the REST API response shapes. These are CLI-side types (not importing from server code to keep CLI decoupled):
     - `TaskResponse`: `{ id: number, title: string, description: string | null, status: string, priority: string, project_id: number, assignee: string | null, created_by: string, due_date: string | null, created_at: string, updated_at: string, tags: string[] }`
     - `ProjectResponse`: `{ id: number, name: string, description: string | null, created_at: string, updated_at: string }`
     - `CreateTaskInput`: `{ title: string, project_id: number, created_by: string, description?: string, priority?: string, assignee?: string, due_date?: string, tags?: string[] }`
     - `UpdateTaskInput`: `{ title?: string, description?: string | null, status?: string, priority?: string, assignee?: string | null, due_date?: string | null, tags?: string[] }`
     - `TaskFilters`: `{ project_id?: number, status?: string, assignee?: string, search?: string, tags?: string, due_before?: string, due_after?: string }`
     - `ApiErrorResponse`: `{ error: string, message: string, details?: unknown }`

5. Create `src/cli/api/client.ts`:
   - Import `env` from `../config/env.js`
   - Import types from `./types.js`
   - Export `ApiClientError` class extending Error with `statusCode: number` and `apiError: ApiErrorResponse` properties.
   - Create private `apiRequest<T>(endpoint: string, options?: RequestInit): Promise<T>` function:
     - Builds full URL: `${env.API_BASE_URL}${endpoint}`
     - Uses native `fetch()` (Node 18+ built-in, no import needed)
     - Sets headers: `Content-Type: application/json`, `X-API-Key: ${env.API_KEY}`, plus any provided headers
     - Adds 10-second timeout via AbortController: `const controller = new AbortController(); const timeout = setTimeout(() => controller.abort(), 10000);` Pass `signal: controller.signal` to fetch, clear timeout in finally block.
     - On non-ok response: parse body as JSON, throw `ApiClientError` with status and parsed error body
     - On network error (fetch throws): catch and throw a new Error with message "Cannot reach API server at {url}. Is it running?"
     - Return `response.json()` cast to `T`
   - Export convenience functions:
     - `createTask(data: CreateTaskInput): Promise<TaskResponse>` -- POST `/api/v1/tasks`
     - `listTasks(filters?: TaskFilters): Promise<TaskResponse[]>` -- GET `/api/v1/tasks` with URLSearchParams from filters (skip undefined values)
     - `getTask(id: number): Promise<TaskResponse>` -- GET `/api/v1/tasks/${id}`
     - `updateTask(id: number, data: UpdateTaskInput): Promise<TaskResponse>` -- PUT `/api/v1/tasks/${id}`
  </action>
  <verify>
    Run `npx tsc --noEmit` to verify TypeScript compilation succeeds with no errors. Verify `node_modules/commander`, `node_modules/cli-table3`, `node_modules/chalk`, and `node_modules/dotenv` exist.
  </verify>
  <done>
    All CLI dependencies installed. API client compiles and exports createTask, listTasks, getTask, updateTask. Env config validates API_KEY presence. Types match REST API contract.
  </done>

1. Create `src/cli/output/formatters.ts`:
   - Import `Table` from `cli-table3` and `chalk` from `chalk`
   - Export `formatStatus(status: string): string` -- color-codes status values:
     - open: chalk.blue, in_progress: chalk.yellow, done: chalk.green, closed: chalk.gray, blocked: chalk.red
     - Unknown status: chalk.white (fallback)
   - Export `formatPriority(priority: string): string` -- color-codes priority values:
     - urgent: chalk.red.bold, high: chalk.red, medium: chalk.yellow, low: chalk.gray
     - Unknown priority: chalk.white (fallback)
   - Export `formatTaskTable(tasks: TaskResponse[]): string`:
     - Create cli-table3 instance with headers: ID, Title, Status, Priority, Assignee, Due Date
     - Use `chalk.bold()` for header text
     - Set `style: { head: [], border: ['gray'] }` (disable cli-table3 default head colors, use chalk instead)
     - Set `wordWrap: true`
     - For each task, push row with: `task.id`, `task.title` (truncate to 45 chars with ellipsis if longer), `formatStatus(task.status)`, `formatPriority(task.priority)`, `task.assignee || '-'`, `task.due_date ? new Date(task.due_date).toLocaleDateString() : '-'`
     - Return `table.toString()`
   - Export `formatTaskDetail(task: TaskResponse): string`:
     - Format a single task for display after create/update operations
     - Show key fields on separate lines with labels:
       ```
       ID:          {id}
       Title:       {title}
       Status:      {formatStatus(status)}
       Priority:    {formatPriority(priority)}
       Project:     {project_id}
       Assignee:    {assignee || '-'}
       Created by:  {created_by}
       Due date:    {due_date formatted or '-'}
       Tags:        {tags.join(', ') || '-'}
       Created:     {created_at formatted}
       ```
     - Use `chalk.bold()` for labels, align values with padding

2. Create `src/cli/output/error-handler.ts`:
   - Import `chalk` from `chalk`
   - Import `ApiClientError` from `../api/client.js`
   - Export `handleError(error: unknown): void`:
     - If `error instanceof ApiClientError`:
       - Print `chalk.red(`Error (${error.statusCode}): ${error.message}`)` to stderr
       - If statusCode === 401: also print `chalk.yellow('Check your API_KEY in .env')` to stderr
       - If statusCode === 404: also print `chalk.yellow('Resource not found')` to stderr
     - Else if `error instanceof Error`:
       - If error.name === 'AbortError': print `chalk.red('Request timed out. Is the API server running?')` to stderr
       - Else: print `chalk.red(`Error: ${error.message}`)` to stderr
     - Else: print `chalk.red('An unexpected error occurred')` to stderr
     - Always set `process.exitCode = 1` (NOT `process.exit(1)` -- allow graceful cleanup)

3. Create `src/cli/bin/tasks.ts`:
   - First line must be: `#!/usr/bin/env node`
   - Import `{ program }` from `commander`
   - Import `{ createCommand }` from `../commands/create.js`
   - Configure program:
     ```
     program
       .name('tasks')
       .description('Wood Fired Bugs - Task management CLI')
       .version('1.0.0');
     ```
   - Add command: `program.addCommand(createCommand);`
   - Parse: `program.parse(process.argv);`
   - Note: list and update commands will be added in Plan 02.

4. Create `src/cli/commands/create.ts`:
   - Import `{ Command }` from `commander`
   - Import `{ createTask }` from `../api/client.js`
   - Import `{ formatTaskDetail }` from `../output/formatters.js`
   - Import `{ handleError }` from `../output/error-handler.js`
   - Import `chalk` from `chalk`
   - Define valid priorities: `['low', 'medium', 'high', 'urgent']`
   - Export `createCommand` as new Command('create'):
     - `.description('Create a new task')`
     - `.requiredOption('-t, --title <title>', 'Task title')`
     - `.requiredOption('-p, --project <id>', 'Project ID', parseInt)`
     - `.requiredOption('-c, --created-by <name>', 'Creator name')`
     - `.option('-d, --description <text>', 'Task description')`
     - `.option('--priority <level>', 'Priority: low, medium, high, urgent', 'medium')`
     - `.option('-a, --assignee <name>', 'Assignee name')`
     - `.option('--due <date>', 'Due date (ISO8601 format, e.g. 2025-12-31T00:00:00Z)')`
     - `.option('--tags <tags>', 'Comma-separated tags')`
   - Action handler (async):
     - Validate priority is in valid list; if not, print error with valid options and set `process.exitCode = 1`, return
     - Validate project ID is a valid number (parseInt may return NaN); if NaN, print error, set exitCode, return
     - Build CreateTaskInput object from options. If `--tags` provided, split by comma and trim each.
     - Call `createTask(input)` in try/catch
     - On success: print `chalk.green('Task created successfully')` then `formatTaskDetail(task)` to stdout
     - On error: call `handleError(error)`

5. Add to `package.json`:
   - Add `"bin"` field: `{ "tasks": "./dist/cli/bin/tasks.js" }`
