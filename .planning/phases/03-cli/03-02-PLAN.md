     - Validate project ID if provided: if NaN after parseInt, print error, set exitCode=1, return.
     - Build filters object of type TaskFilters from provided options. Only include properties that were actually provided (skip undefined). Map `--tags` to comma-separated string (API expects `tags=bug,ui` as a query param string).

   - Import `{ formatTaskDetail }` from `../output/formatters.js`
   - Import `{ handleError }` from `../output/error-handler.js`

     - `.argument('<id>', 'Task ID to update')`
     - `.option('-t, --title <title>', 'New title')`
     - `.option('-d, --description <text>', 'New description')`
     - `.option('-s, --status <status>', 'New status')`
     - `.option('--priority <level>', 'New priority')`
     - `.option('-a, --assignee <name>', 'New assignee')`

     - On success: print `chalk.green(`Task #${task.id} updated successfully`)` then `formatTaskDetail(task)`
     - On error: call `handleError(error)`

Test all commands by mocking the API client module and verifying command behavior. Use Vitest's `vi.mock()` to mock `../api/client.js` so tests don't need a running server.

**Important testing pattern:** Since Commander.js calls `process.exit()` on certain errors (like missing required options), and since the env module calls `process.exit(1)` when API_KEY is missing, tests need to:
- Set `process.env.API_KEY = 'test-key'` and `process.env.API_BASE_URL = 'http://localhost:3000'` before any imports that trigger env loading.
- Use `vi.mock('../api/client.js')` at the top of each test file to mock ALL API calls.
- Mock `console.log` and `console.error` with `vi.spyOn()` to capture and assert output.
- Create a fresh Command for each test, add the command being tested, and call `program.parseAsync(['node', 'test', ...args])`.

**Important:** The env.ts module validates API_KEY at import time. Mock it or set the env var BEFORE the dynamic import. Use `vi.mock('../config/env.js', () => ({ env: { API_BASE_URL: 'http://localhost:3000', API_KEY: 'test-key' } }))` to avoid the validation check.
