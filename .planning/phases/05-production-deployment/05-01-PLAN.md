1. Imports `createServer` from `./server.js`
2. Reads PORT (default 3000) and HOST (default '0.0.0.0') from environment
3. Calls `createServer()`, then `server.listen({ port, host })`
4. Registers SIGTERM and SIGINT handlers that:
   - Log the received signal via `server.log.info({ signal }, 'Received shutdown signal')`
   - Call `await server.close()` to stop accepting connections and drain existing
   - Call `app.db.close()` to cleanly close the SQLite database
   - Log 'Shutdown complete' and call `process.exit(0)`
5. Registers `uncaughtException` and `unhandledRejection` handlers that log via `server.log.fatal()` and exit with code 1
6. Logs startup info: host, port, NODE_ENV

Remove the `startServer()` function from `src/api/server.ts` -- it is superseded by `start.ts`. The `createServer()` factory stays (used by tests and start.ts).

Add npm scripts to `package.json`:
- `"start": "node dist/api/start.js"` (production start)
- `"dev": "tsx src/api/start.ts"` (development start with pino-pretty)

The HOST default of '0.0.0.0' means the server binds to all IPv4 interfaces, making it reachable from other LAN machines. This satisfies INFRA-04.

Important: Do NOT use `trustProxy` (no reverse proxy in this deployment). Do NOT change the existing Pino logger configuration in createServer -- it already outputs JSON in production and pino-pretty in development.

Create `deploy/` directory with three files:

**deploy/wood-fired-bugs.service** -- systemd unit file:
```ini
[Unit]
Description=Wood Fired Bugs Task Tracker
After=network.target
StartLimitBurst=3
StartLimitIntervalSec=60

[Service]
Type=simple
User=stuart
Group=stuart
WorkingDirectory=/opt/wood-fired-bugs
EnvironmentFile=/opt/wood-fired-bugs/.env
ExecStart=/usr/bin/node /opt/wood-fired-bugs/dist/api/start.js
Restart=on-failure
RestartSec=5s
TimeoutStopSec=15s

# Security hardening
ProtectSystem=strict
ProtectHome=read-only
PrivateTmp=yes
NoNewPrivileges=yes
ReadWritePaths=/opt/wood-fired-bugs/data /opt/wood-fired-bugs/backups

# Graceful shutdown
KillMode=mixed
KillSignal=SIGTERM

# Logging to journald
StandardOutput=journal
StandardError=journal
SyslogIdentifier=wood-fired-bugs

[Install]
WantedBy=multi-user.target
```

Key decisions per research:
- `StartLimitBurst` and `StartLimitIntervalSec` in [Unit] section (NOT [Service] -- silently ignored there per pitfall #1)
- `Restart=on-failure` NOT `always` (allows manual stop without restart loop per pitfall #8)
- `ProtectHome=read-only` NOT `yes` (service may need to read ~/.npmrc or node paths)
- `ReadWritePaths` includes both data and backups directories
- `TimeoutStopSec=15s` gives graceful shutdown handler time to drain connections and close DB

**deploy/install.sh** -- deployment setup script:
```bash
#!/usr/bin/env bash
set -euo pipefail

# Wood Fired Bugs - Deployment Setup
# Run as root or with sudo: sudo bash deploy/install.sh
#
# Prerequisites:
#   - Node.js installed at /usr/bin/node
#   - Project built (npm run build)
#   - sqlite3 CLI installed (sudo apt-get install sqlite3)
#
# What this script does:
#   1. Creates /opt/wood-fired-bugs directory structure
#   2. Copies built application files
#   3. Installs production dependencies
#   4. Copies systemd unit file
#   5. Enables and starts the service
#
# After running, you MUST:
#   - Edit /opt/wood-fired-bugs/.env to set real API_KEYS
#   - Run: sudo systemctl restart wood-fired-bugs

INSTALL_DIR="/opt/wood-fired-bugs"
SOURCE_DIR="$(cd "$(dirname "$0")/.." && pwd)"
SERVICE_USER="stuart"

echo "=== Wood Fired Bugs Deployment ==="

# Create directory structure
mkdir -p "$INSTALL_DIR"/{data,backups,dist}

# Copy application files
cp -r "$SOURCE_DIR/dist/"* "$INSTALL_DIR/dist/"
cp "$SOURCE_DIR/package.json" "$INSTALL_DIR/"
cp "$SOURCE_DIR/package-lock.json" "$INSTALL_DIR/" 2>/dev/null || true

# Install production dependencies only
cd "$INSTALL_DIR"
npm install --omit=dev

# Copy env template if .env doesn't exist
if [ ! -f "$INSTALL_DIR/.env" ]; then
  cp "$SOURCE_DIR/deploy/wood-fired-bugs.env.example" "$INSTALL_DIR/.env"
  echo "WARNING: Created .env from template. Edit $INSTALL_DIR/.env to set API_KEYS before starting."
fi

# Set ownership
chown -R "$SERVICE_USER:$SERVICE_USER" "$INSTALL_DIR"

# Install systemd unit
cp "$SOURCE_DIR/deploy/wood-fired-bugs.service" /etc/systemd/system/
systemctl daemon-reload
systemctl enable wood-fired-bugs

echo ""
echo "=== Deployment Complete ==="
echo "Next steps:"
echo "  1. Edit $INSTALL_DIR/.env (set API_KEYS)"
echo "  2. sudo systemctl start wood-fired-bugs"
echo "  3. sudo systemctl status wood-fired-bugs"
echo "  4. sudo journalctl -u wood-fired-bugs -f"
```

Make install.sh executable: `chmod +x deploy/install.sh`
