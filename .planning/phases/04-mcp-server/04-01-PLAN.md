  <files>package.json, src/mcp/errors.ts</files>
  <action>
1. Install the MCP SDK:
   ```
   npm install @modelcontextprotocol/sdk
   ```
   Zod 4.x is already installed (^4.3.6 in package.json) so no Zod changes needed.

2. Create `src/mcp/errors.ts` -- error conversion utility that maps Phase 1 custom errors to McpError:
   - Import `McpError` and `ErrorCode` from `@modelcontextprotocol/sdk/types.js`
   - Import `ValidationError`, `NotFoundError`, `BusinessError` from `../services/errors.js`
   - Export function `convertToMcpError(error: unknown): McpError` that:
     - `ValidationError` -> `McpError(ErrorCode.InvalidParams, "Validation failed", { fieldErrors: error.fieldErrors })`
     - `NotFoundError` -> `McpError(ErrorCode.InvalidRequest, error.message, { entity: error.entity, id: error.id })`
     - `BusinessError` -> `McpError(ErrorCode.InvalidRequest, error.message)`
     - Unknown errors -> log full error to `console.error()`, return `McpError(ErrorCode.InternalError, "An internal error occurred")`
   - IMPORTANT: McpError constructor signature is `new McpError(code, message, data?)`. The `data` parameter is optional and accepts a plain object.
   - IMPORTANT: Check actual SDK exports -- McpError and ErrorCode may be exported from `@modelcontextprotocol/sdk/types.js` or another subpath. Verify by checking `node_modules/@modelcontextprotocol/sdk` exports before writing imports.

3. Add npm scripts to package.json:
   - `"mcp:start": "node dist/mcp/index.js"` -- runs compiled MCP server
   - `"mcp:dev": "tsx src/mcp/index.ts"` -- runs MCP server in dev mode
  </action>
  <verify>
   - `npm ls @modelcontextprotocol/sdk` shows installed version
   - `npx tsc --noEmit src/mcp/errors.ts` compiles without errors (run after Task 2 creates all files, or test in isolation)
  </verify>
  <done>MCP SDK installed, error conversion utility compiles and handles all 4 error types (ValidationError, NotFoundError, BusinessError, unknown)</done>

<task type="auto">
  <name>Task 2: Create MCP tool handlers, server factory, and stdio entry point</name>
  <files>src/mcp/tools/task-tools.ts, src/mcp/server.ts, src/mcp/index.ts</files>
  <action>
1. Create `src/mcp/tools/task-tools.ts` -- tool registration function:
   - Import `McpServer` from `@modelcontextprotocol/sdk/server/mcp.js`
   - Import `TaskService` from `../../services/task.service.js` and `ProjectService` from `../../services/project.service.js`
   - Import shared schemas: `CreateTaskSchema`, `UpdateTaskSchema`, `TaskFiltersSchema` from `../../schemas/task.schema.js`
   - Import `z` from `zod`
   - Import `convertToMcpError` from `../errors.js`
   - Export function `registerTaskTools(server: McpServer, taskService: TaskService, projectService: ProjectService): void`

   Register 5 tools using `server.registerTool()`:

   **create_task:**
   - description: "Create a new task in a project"
   - inputSchema: `CreateTaskSchema` (shared from Phase 1)
   - Handler: call `taskService.createTask(input)`, return text content with task title/ID/status, and structuredContent with full task object
   - Wrap in try/catch, throw `convertToMcpError(error)` on failure

   **get_task:**
   - description: "Get a task by its ID"
   - inputSchema: `z.object({ id: z.number().int().positive() })`
   - Handler: call `taskService.getTask(id)`, return text with title/status/priority/description summary, structuredContent with full task
   - Wrap in try/catch

   **update_task:**
   - description: "Update an existing task by ID. Can update title, description, status, priority, assignee, due_date, and tags."
   - inputSchema: `z.object({ id: z.number().int().positive(), updates: UpdateTaskSchema })`
   - Handler: call `taskService.updateTask(id, updates)`, return text confirming update, structuredContent with updated task
   - Wrap in try/catch

   **list_tasks:**
   - description: "List tasks with optional filters (project_id, status, assignee, tags, due_before, due_after, search)"
   - inputSchema: `TaskFiltersSchema` (shared, already has all fields optional via .partial())
   - Handler: call `taskService.listTasks(filters)`, return text summary with count and bullet list of tasks (id, title, status, priority), structuredContent with array
   - Handle empty results: return text "No tasks found matching filters."
   - Wrap in try/catch

   **delete_task:**
   - description: "Delete a task by its ID"
   - inputSchema: `z.object({ id: z.number().int().positive() })`
   - Handler: call `taskService.deleteTask(id)`, return text confirming deletion
   - Wrap in try/catch

   IMPORTANT: Do NOT use console.log() anywhere in MCP code -- use console.error() for any logging. stdout is reserved for JSON-RPC messages in stdio transport.

   IMPORTANT: The service methods are synchronous (they use better-sqlite3 which is sync). The tool handlers are async functions. Just call service methods directly -- no await needed on service calls themselves.

   IMPORTANT: Verify the exact `server.registerTool()` signature by checking SDK types. The research shows it takes (name, config, handler) where config has `description` and `inputSchema`. The handler receives the parsed input directly. Confirm this pattern matches the installed SDK version.

2. Create `src/mcp/server.ts` -- server factory:
   - Import `McpServer` from `@modelcontextprotocol/sdk/server/mcp.js`
   - Import `registerTaskTools` from `./tools/task-tools.js`
   - Import `TaskService` and `ProjectService` types
   - Export function `createMcpServer(taskService: TaskService, projectService: ProjectService): McpServer`
   - Creates `new McpServer({ name: "wood-fired-bugs", version: "1.0.0" })`
   - Calls `registerTaskTools(server, taskService, projectService)`
   - Returns the server instance
   - This factory pattern allows tests to create servers without stdio transport

3. Create `src/mcp/index.ts` -- stdio entry point:
   - Import `StdioServerTransport` from `@modelcontextprotocol/sdk/server/stdio.js`
   - Import `createApp` from `../index.js`
   - Import `createMcpServer` from `./server.js`
   - Define async `main()`:
     - Determine DB path: `process.env.DB_PATH || './data/tasks.db'`
     - Call `const app = await createApp(dbPath)`
     - Call `const server = createMcpServer(app.taskService, app.projectService)`
     - Create `const transport = new StdioServerTransport()`
     - Call `await server.connect(transport)`
     - `console.error("Wood Fired Bugs MCP Server running on stdio")`
   - Call `main().catch(error => { console.error("Fatal error:", error); process.exit(1); })`
   - Add process-level error handlers:
     ```
     process.on('uncaughtException', (error) => { console.error('Uncaught exception:', error); process.exit(1); });
     process.on('unhandledRejection', (reason) => { console.error('Unhandled rejection:', reason); process.exit(1); });
     ```

4. After creating all files, run `npx tsc --noEmit` to verify the entire project compiles without type errors.
