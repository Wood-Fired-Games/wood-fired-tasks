  <name>Task 1: Create backup command implementation</name>
  <files>src/cli/commands/backup.ts</files>
  <action>
Create `src/cli/commands/backup.ts` implementing the `tasks backup` CLI command.

This command is UNIQUE among all CLI commands: it does NOT use the HTTP API client (`src/cli/api/client.ts`). Instead, it opens the SQLite database file directly using better-sqlite3 and calls `db.backup(destPath)`.

Implementation details:
- Import `Command` from 'commander', `Database` from 'better-sqlite3', `resolve` and `dirname` from 'path', `existsSync` and `mkdirSync` and `statSync` from 'fs', `chalk` from 'chalk'
- Import `jsonOutput` from '../output/json-output.js' and `shouldUseColor` from '../output/formatters.js'
- Export `backupCommand` as `new Command('backup')`
- Description: 'Create a SQLite backup of the task database'
- Option: `-o, --output <path>` with description 'Backup destination path'
- Default output path: `./tasks-backup-{timestamp}.db` where timestamp is `new Date().toISOString().replace(/[:.]/g, '-')`
- The action handler must be async (because `db.backup()` returns a Promise)
- Read `DATABASE_PATH` from `process.env.DATABASE_PATH`, falling back to `'./data/tasks.db'`. The CLI env module (`src/cli/config/env.ts`) already loads dotenv, so `process.env` will have .env values. Import `'../config/env.js'` at the top (side-effect import) to ensure dotenv is loaded before accessing process.env.
- Resolve the output path to absolute using `resolve(options.output)`
- Before calling backup, verify the output directory exists. If not, create it with `mkdirSync(dirname(destPath), { recursive: true })`
- Open source database with `new Database(dbPath, { readonly: true })` -- readonly to avoid write lock conflicts with the running server
- Call `await db.backup(destPath)` in a try/finally block that always calls `db.close()`
- On success: check JSON mode via `program.optsWithGlobals().json`, output accordingly
  - Terminal mode: print success message with chalk.green, include the backup file path and size (use `statSync(destPath).size` formatted as KB/MB)
  - JSON mode: call `jsonOutput({ path: destPath, size: statSync(destPath).size, source: dbPath })`
- On error: catch errors and print descriptive messages
  - If source DB does not exist: print "Database not found at {dbPath}" and set `process.exitCode = 1`
  - For other errors: print error message with chalk.red and set `process.exitCode = 1`

Do NOT add a REST API endpoint for backup. Do NOT use `VACUUM INTO` -- use `db.backup()` which is the canonical safe approach for WAL-mode hot backups.
